# Анализ заказов интернет-магазина

Учебный проект для пакетной обработки CSV-файлов с заказами.

Скрипт просматривает все CSV-файлы в папке `data`, выбирает заказы со статусом `Delivered` и формирует сводный отчёт по каждому файлу.

## Результат обработки

Для каждого файла рассчитываются:

- `total_revenue` — сумма `total_amount` по доставленным заказам;
- `average_order_value` — среднее значение `total_amount` по доставленным заказам;
- `order_count` — количество доставленных заказов.

Итоговый отчёт сохраняется в:

```text
reports/summary_report.csv
```

Ошибки чтения и проверки файлов сохраняются в:

```text
logs/errors.log
```

Если один из файлов повреждён, пуст или не соответствует ожидаемой структуре, он пропускается. Обработка остальных файлов продолжается.

## Структура проекта

```text
order_analysis/
├── data/                  # исходные CSV-файлы
├── reports/               # итоговый отчёт
├── logs/                  # журнал ошибок
├── src/
│   ├── __init__.py
│   └── analyzer.py        # класс OrderAnalyzer
├── config.py              # настройки проекта
├── run.py                 # точка запуска
├── requirements.txt
└── README.md
```

## Логика работы

1. `run.py` создаёт экземпляр класса `OrderAnalyzer`.
2. Класс находит все файлы с расширением `.csv` в папке `data`.
3. Каждый файл читается и проверяется:
   - файл не должен быть пустым;
   - должны присутствовать обязательные колонки;
   - `total_amount` должен содержать числовые значения.
4. Из корректного файла выбираются строки, где `status == "Delivered"`.
5. Рассчитываются выручка, средний чек и количество заказов.
6. Результаты по всем корректным файлам объединяются в один отчёт.
7. Ошибочные файлы записываются в лог и не останавливают выполнение программы.

## Запуск

Требуется Python 3.8 или выше.

Установка зависимости:

```bash
pip install -r requirements.txt
```

Запуск:

```bash
python run.py
```

Пример вывода:

```text
Обработано файлов: 2
Файлов с ошибками: 1
Итоговый отчёт: .../reports/summary_report.csv
```

## Проверка работы

В папке `data` находятся два корректных файла:

```text
order_10000.csv
order_100000.csv
```

Также добавлен файл `broken_order.csv` с нечисловым значением в колонке `total_amount`. Он используется для проверки обработки ошибок.

После запуска в `reports/summary_report.csv` должны быть две строки:

| file_name | total_revenue | average_order_value | order_count |
|---|---:|---:|---:|
| order_10000.csv | 1069477.83 | 754.75 | 1417 |
| order_100000.csv | 10874402.61 | 761.14 | 14287 |

Файл `broken_order.csv` в отчёт не попадает. В `logs/errors.log` появляется запись о том, что колонка `total_amount` содержит нечисловые значения.

Таким образом проверяются сразу два сценария:

- корректная пакетная обработка нескольких файлов;
- продолжение работы после ошибки в одном из файлов.

### Контрольный тест расчётов

Для отдельной проверки формул можно временно добавить в `data` файл `test_orders.csv`:

```csv
order_id,person_id,order_date,status,total_amount,currency,payment_method,shipping_method,notes
1,101,2026-01-10,Delivered,100.00,USD,credit_card,standard,
2,102,2026-01-11,Delivered,300.00,USD,paypal,express,
3,103,2026-01-12,Cancelled,900.00,USD,credit_card,standard,
```

После запуска для этого файла должна появиться строка:

```text
test_orders.csv,400.0,200.0,2
```

Заказ со статусом `Cancelled` в расчёт не входит. После проверки временный файл можно удалить.

## Основные файлы

### `config.py`

Содержит пути, имена выходных файлов, название колонки статуса, значение статуса для фильтрации и список обязательных колонок.

### `src/analyzer.py`

Содержит класс `OrderAnalyzer` и методы:

- `load_file` — чтение и проверка CSV;
- `filter_delivered_orders` — фильтрация доставленных заказов;
- `calculate_metrics` — расчёт метрик;
- `process_file` — обработка одного файла;
- `process_all_files` — обработка всех CSV и сохранение общего отчёта.

### `run.py`

Содержит функцию `main`, создаёт объект `OrderAnalyzer`, запускает обработку и выводит итоговые счётчики.
